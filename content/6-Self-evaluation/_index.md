During my internship in the **Workforce Bootcamp - First Cloud AI Journey** program, I had the opportunity to learn and practice topics related to AWS, Cloud Architecture, DevOps, and AI through a real project context: the **Netflop** movie streaming web system.

By studying Netflop's current architecture, I improved my ability to understand system design, analyze the role of each AWS service, document the media processing flow, identify security risks, and organize technical information into a structured report. The main areas I worked on included EC2/Nginx, React frontend, Node.js backend API, RDS MySQL, S3, MediaConvert, CloudFront, Lambda, EventBridge, CloudWatch, SNS, IAM, and Systems Manager.

In terms of work attitude, I tried to complete the report content week by week, actively recorded new knowledge, compared technical information with the project notes, and revised the report so it matched the perspective of an internship student. Besides the results achieved, I also realized that I need to improve my technical depth, system testing skills, and ability to present information more concisely.

### Achievements

* Understood how a real website is deployed on AWS instead of viewing each service separately.
* Described the Netflop architecture, including web layer, backend API, database, media storage, video processing, CDN, and automation.
* Understood the video upload flow from backend to S3, processing with MediaConvert, and HLS delivery through CloudFront.
* Understood how EventBridge and Lambda automate video processing status updates.
* Improved awareness of cloud security, especially IAM, security groups, S3 access, RDS access, and signed cookies.
* Practiced technical reporting, weekly worklog writing, and summarizing lessons learned from events.

### Strengths

* Strong self-learning attitude and willingness to read more documentation when facing new AWS services.
* Ability to connect theoretical knowledge with a real web architecture.
* Organized work through a clear 12-week worklog.
* Ability to identify system limitations and propose improvements instead of only describing results.

### Areas For Improvement

* Need more hands-on practice with production deployment, CI/CD, and real cloud troubleshooting.
* Need to improve log analysis, CloudWatch usage, and detailed testing checklist design.
* Need to present reports more concisely and avoid repeating content.
* Need to improve technical communication, especially when explaining architecture to people who are not familiar with the system.

### Future Direction

After the internship, I want to continue learning more about AWS Solutions Architecture, DevOps automation, cloud security, and applying AI to system operations. I also want to practice more with CI/CD, Infrastructure as Code, observability, and safer deployment patterns for real web applications.
